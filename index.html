Maldo API Documentation

1. TestConnection

Endpoint: /api/testconnection

Metoda: POST

Opis: Ova funkcija služi za testiranje konekcije sa bazom podataka. Ukoliko je konekcija uspešna, vraća poruku "Connection successful". U suprotnom, vraća poruku sa opisom greške.

Odgovori:

  • 200 OK: Konekcija je uspešna. Vraća poruku "Connection successful".
  • 500 Internal Server Error: Greška prilikom konekcije. Vraća detaljan opis greške.

Primer korišćenja:

POST /api/testconnection HTTP/1.1
Host: [domen]

Odgovor:

{
    "message": "Connection successful"
}

2. AddEmployer

Endpoint: /api/Employer/Add

Metoda: POST

Opis: Ova funkcija dodaje novog poslodavca u bazu podataka. Funkcija očekuje JSON objekat koji sadrži korisničko ime, lozinku, email, naziv firme i broj telefona poslodavca.

Telo zahteva (Body):

  • username (string): Korisničko ime poslodavca. (Obavezno)
  • password (string): Lozinka poslodavca. (Obavezno)
  • email (string): Email adresa poslodavca. (Obavezno)
  • name (string): Naziv firme poslodavca. (Obavezno)
  • phone (string): Broj telefona poslodavca. (Obavezno)

Odgovori:

  • 200 OK: Poslodavac je uspešno dodat. Vraća poruku "Success".
  • 400 Bad Request: Nevalidan unos ili nedostaju neka od obaveznih polja. Vraća poruku sa opisom greške.

Primer korišćenja:

POST /api/Employer/Add HTTP/1.1
Host: [domen]
Content-Type: application/json

{
    "username": "exampleUser",
    "password": "examplePass",
    "email": "example@example.com",
    "name": "Example Company",
    "phone": "123456789"
}

Odgovor:

{
    "message": "Success"
}

Napomena: Ukoliko korisničko ime već postoji u bazi, funkcija će vratiti status 400 sa porukom "Username already exists".

3. UpdateEmployer

Endpoint: /api/Employer/{id}

Metoda: PUT

Opis: Ova funkcija služi za ažuriranje podataka o poslodavcu u bazi podataka. Korisnik može ažurirati korisničko ime, lozinku, email, naziv firme i broj telefona poslodavca.

Parametri:

  • id (int): Jedinstveni identifikator poslodavca koji se ažurira. (Obavezno)

Telo zahteva (Body):

  • username (string): Novo korisničko ime poslodavca. (Opcionalno)
  • password (string): Nova lozinka poslodavca. (Opcionalno)
  • email (string): Novi email poslodavca. (Opcionalno)
  • name (string): Novi naziv firme poslodavca. (Opcionalno)
  • phone (string): Novi broj telefona poslodavca. (Opcionalno)

Odgovori:

  • 200 OK: Podaci o poslodavcu su uspešno ažurirani. Vraća poruku "Employer successfully updated."
  • 400 Bad Request: Nevalidan unos ili nedostaju podaci. Vraća poruku sa opisom greške.
  • 500 Internal Server Error: Greška prilikom ažuriranja podataka. Vraća detaljan opis greške.

Primer korišćenja:

        PUT /api/Employer/1 HTTP/1.1
        Host: [domen]
        Content-Type: application/json
        
        {
          "username": "newUsername",
          "password": "newPassword",
          "email": "newEmail@example.com",
          "name": "New Company Name",
          "phone": "987654321"
        }

Odgovor:

        {
          "message": "Employer successfully updated."
        }

Telo zahteva (Body):

  • 500 Internal Server Error: Greška prilikom dodavanja ocene. Vraća detaljan opis greške.

Primer korišćenja:

        POST /api/EmployerRating/Add HTTP/1.1
        Host: [domen]
        Content-Type: application/json
        
        {
          "employerId": 1,
          "driverId": 2,
          "rating": 5,
          "comment": "Great experience!"
        }

Odgovor:

        {
          "message": "Rating successfully added."
        }
